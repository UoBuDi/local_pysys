# 拆分匹配功能开发计划

## 1. 需求概述

### 1.1 业务需求
在check_data数据库中，存在以"_yc"结尾的核查数据表和以"cf"开头的拆分数据表。需要实现将核查数据表中的记录根据"通行标识ID"与拆分数据表进行匹配，匹配成功则在该核查数据表的"备注"字段标注"已拆回"，匹配不成功则标注"未拆回"。

### 1.2 功能需求
1. 在系统工具菜单下新增"拆分匹配"菜单
2. 点击"拆分匹配"后显示下拉选择框，列出check_data数据库中所有以"_yc"结尾的数据表
3. 选择数据表后，在表格中显示匹配结果
4. 每页显示20条数据
5. 支持根据字段名进行筛选

### 2.3 关联分析

**关联字段**：
- 核查表：通行标识ID（varchar(50)）
- 拆分表：通行标识ID（varchar(40)）

**关联逻辑**：
1. 通过核查表的"通行标识ID"与拆分表的"通行标识ID"进行匹配
2. 匹配成功 → 在核查表"备注"字段标注"已拆回"
3. 匹配失败 → 在核查表"备注"字段标注"未拆回"

**性能考虑**：
- 拆分表在"通行标识ID"字段上已有索引
- 建议在核查表"通行标识ID"字段添加索引以提高查询效率

#### 5.2.2 脚本逻辑

```typescript
<script setup lang="ts">
import { ref, onMounted } from 'vue'
import { getSplitMatchTables, getSplitMatchData, executeMatch } from '@/api/splitMatch'

// 响应式数据
const tableList = ref<string[]>([])
const selectedTable = ref('')
const tableData = ref([])
const columns = ref<Column[]>([])
const fieldFilter = ref('')
const currentPage = ref(1)
const pageSize = ref(20)
const total = ref(0)

// 生命周期
onMounted(async () => {
  await loadTableList()
})

// 方法
const loadTableList = async () => {
  const res = await getSplitMatchTables()
  tableList.value = res.data
}

const handleSearch = async () => {
  await loadTableData()
}

const loadTableData = async () => {
  const res = await getSplitMatchData({
    table_name: selectedTable.value,
    field_filter: fieldFilter.value,
    page: currentPage.value,
    page_size: pageSize.value
  })
  tableData.value = res.data.list
  total.value = res.data.total
  columns.value = generateColumns(tableData.value[0])
}

const handleExecuteMatch = async () => {
  if (!selectedTable.value) {
    ElMessage.warning('请先选择数据表')
    return
  }
  const res = await executeMatch(selectedTable.value)
  ElMessage.success(`匹配完成！已拆回：${res.data.matched_count}，未拆回：${res.data.unmatched_count}`)
  await loadTableData()
}
</script>
```
