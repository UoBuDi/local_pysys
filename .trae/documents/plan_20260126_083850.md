# 拆分匹配功能开发计划

## 1. 需求概述
在系统工具菜单下新增"拆分匹配"功能，实现check_data数据库中_yc表与cf表的匹配，根据通行标识ID进行匹配并标注备注字段。

## 2. 数据库分析结论
- **2024-01_yc表**：871行数据，核心字段包括通行标识ID、车牌号码、备注等
- **202005-202311_cf_1215表**：58,337,757行数据，通行标识ID字段有索引
- **可匹配的_yc表**：共22个（2024-01至2025-10）

## 3. 开发任务

### 3.1 后端开发
1. 添加check_data数据库配置到config.ini
2. 创建split_match_service.py服务类
3. 创建routers/split_match.py路由模块
4. 实现3个API接口：
   - GET /api/split-match/tables（获取_yc表列表）
   - GET /api/split-match/data（获取表数据，支持分页和筛选）
   - POST /api/split-match/execute（执行匹配）

### 3.2 前端开发
1. 创建views/systemTools/SplitMatch.vue页面组件
2. 创建api/splitMatch.ts接口封装
3. 实现下拉选择框（显示22个_yc表）
4. 实现数据表格（每页20条，支持字段筛选）
5. 实现执行匹配按钮功能

### 3.3 菜单配置
在系统工具菜单下新增"拆分匹配"菜单，路由：/system/split-match

## 4. 实施步骤
1. 后端API开发（约4小时）
2. 前端页面开发（约6小时）
3. 菜单配置与集成（约2小时）
4. 测试与调试（约4小时）
